# Secure Cloud Platform

A full-stack secure cloud data management platform built for a Master's dissertation. It combines **AES-256-GCM encrypted storage** (MinIO), **immutable blockchain audit trails** (Polygon Mumbai), and **real-time AI anomaly detection** (4-model ensemble) into a single cohesive system.

---

## Architecture

```
Browser (Next.js 14)
       │
       ▼
  API Gateway (Flask :5000)
  ┌────┴─────────────────────┐
  │                          │
Storage Service         Blockchain Service    AI Detection Service
(Flask :5001)           (Flask :5002)         (Flask :5003)
MinIO + AES-256-GCM     Polygon Mumbai        TF Autoencoder
                        Solidity contract     LSTM, IF, RF
                                              Redis event store
```

## Quick Start (Docker)

### Step 1 — Clone and configure environment

```bash
git clone <repo-url> secure-cloud-platform
cd secure-cloud-platform

cp .env.example .env
cp backend/.env.example backend/.env
cp blockchain/.env.example blockchain/.env
cp frontend/.env.local.example frontend/.env.local
```

Edit `backend/.env` and set:
- `PRIVATE_KEY` — your Polygon Mumbai wallet private key (with test MATIC)
- `CONTRACT_ADDRESS` — leave blank until after Step 3

### Step 2 — Train AI models (optional but recommended)

Download the [UNSW-NB15 dataset](https://research.unsw.edu.au/projects/unsw-nb15-dataset) and place:
- `backend/data/unsw_nb15/UNSW_NB15_training-set.csv`
- `backend/data/unsw_nb15/UNSW_NB15_testing-set.csv`

```bash
cd backend
pip install -r requirements.txt
python train_models_unsw.py
cd ..
```

### Step 3 — Deploy smart contract

```bash
cd blockchain
npm install
npx hardhat compile
npx hardhat run scripts/deploy.js --network mumbai
```

Copy the deployed address into `backend/.env` → `CONTRACT_ADDRESS=0x...`

### Step 4 — Start all services

```bash
docker-compose up --build
```

### Step 5 — Open the platform

| Service | URL |
|---------|-----|
| Frontend | http://localhost:3000 |
| API Gateway | http://localhost:5000 |
| Storage Service | http://localhost:5001 |
| Blockchain Service | http://localhost:5002 |
| AI Detection | http://localhost:5003 |
| MinIO Console | http://localhost:9001 |

## Development Without Docker

### Backend

```bash
cd backend
python -m venv .venv
source .venv/bin/activate       # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env            # edit with your credentials

# Start each service in a separate terminal
python storage_service.py       # :5001
python blockchain_service.py    # :5002
python ai_detection_service.py  # :5003
python main.py                  # :5000 (gateway)
```

### Frontend

```bash
cd frontend
npm install
cp .env.local.example .env.local   # edit with your credentials
npm run dev                         # http://localhost:3000
```

### Blockchain

```bash
cd blockchain
npm install
npx hardhat node                    # local hardhat node
npx hardhat run scripts/deploy.js --network localhost
npx hardhat test                    # run test suite
```

## AI Detection Models

The system uses a weighted ensemble of 4 models trained on the UNSW-NB15 intrusion detection dataset:

| Model | Weight | Type |
|-------|--------|------|
| Autoencoder | 30% | Unsupervised anomaly detection |
| Isolation Forest | 25% | Unsupervised outlier detection |
| LSTM | 25% | Sequential pattern recognition |
| Random Forest | 20% | Supervised binary classification |

**Risk levels:**

| Level | Score | Action |
|-------|-------|--------|
| CRITICAL | ≥ 0.8 | BLOCK |
| HIGH | 0.6–0.8 | ALERT |
| MEDIUM | 0.4–0.6 | LOG |
| NORMAL | < 0.4 | PASS |

## Security Features

- **AES-256-GCM** encryption per chunk with PBKDF2 key derivation (600,000 iterations)
- **Random salt + nonce** prepended to every chunk object
- **Immutable audit trail** stored on Polygon blockchain
- **4-model AI ensemble** detecting upload/download anomalies in real time
- **Permission-based access control** (NONE/READ/WRITE/FULL) on-chain
- **Chunked storage** — 10MB chunks prevent single-object compromise
